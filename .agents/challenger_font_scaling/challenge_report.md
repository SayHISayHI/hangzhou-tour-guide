# 字体缩放布局兼容测试挑战报告 (Challenge Report)

## Challenge Summary

**Overall risk assessment**: HIGH (高风险)

在本项测试中，我们针对杭州导游项目的字体字号动态缩放机制（0.8倍至1.5倍）进行了对抗性的布局与样式推演。经过深入审查代码和样式（包括 `rem`, `em`, `px` 混合单位），我们发现在最大缩放（1.5x）和最小缩放（0.8x）的极端边界下，当前重构的样式存在多处严重的视觉崩塌、排版错位及无障碍访问失效漏洞。

---

## 🔒 核心漏洞发现 (Confirmed Vulnerabilities)

### 1. 【高危】卡片标题与描述字号层级倒挂
- **现象描述**：
  - 在 `src/index.css` 中，`.poi-card-info p` (描述) 的字号是动态缩放的：`font-size: calc(0.9rem * var(--text-scale))`。当缩放至最大 1.5 倍时，其字号为 `1.35rem`。
  - 然而，卡片标题 `.poi-card-info h3` 的字号是静态硬编码的：`font-size: 1.3rem`，未参与 `--text-scale` 的缩放计算。
  - **冲突结果**：在 1.5 倍缩放状态下，**描述文字字号 (1.35rem) 竟然大于标题字号 (1.3rem)**！这直接违背了最基本的视觉层级设计，导致标题不再突出，界面布局产生严重的结构性滑稽感。
  - 同样地，在首页 `.section-card` 中，描述 `p` 放大至 `1.65rem`，而标题 `h2` 固定为 `2rem`。这使两者字号对比度从 `1.82倍` 锐减至 `1.21倍`，字号视觉差异几乎消失。

### 2. 【高危】折叠容器固定高度导致文字物理截断与首字下沉错位
- **现象描述**：
  - 在 `src/components/ExpandableText.jsx` 中，段落折叠容器被硬编码了固定高度：`animate={{ height: expanded ? 'auto' : '200px' }}`。
  - 在 1.5 倍缩放状态下，正文字号放大至 `1.65rem` (约 `26.4px`，以 root 16px 计)，配合行高 `1.85` (单行高度 `48.84px`) 与段距下边距 `1.5rem` (24px)。**仅仅一行半的文本加上段距就会消耗掉近 100px 的高度**。
  - **冲突结果**：折叠容器在 200px 的高度内，连一个 3 行以上的段落都无法完整显示，这会导致**折叠态下段落正文在物理上被强行裁切**，出现只剩半截字、多段内容只显示第一段半的严重视觉破缺。
  - 此外，首字下沉 `.drop-cap > p:first-of-type::first-letter` 的字号固定为 `3.5rem` (56px) 且具有 `line-height: 0.8`。在正文放大至 26.4px 时，首字没有跟着一起放大，使其比例失调（原本是 3.2 倍，现在缩水到 2.1 倍）。同时，硬编码的 line-height 与 padding-top 导致首字与放大的第二行文字产生极其难看的基线重叠与错位。

### 3. 【中危】折叠遮罩 `.fade-mask` 产生“脏色块拼接缝”
- **现象描述**：
  - 渐隐遮罩 `.fade-mask` 的背景渐变在 `src/index.css` 中被声明为 `linear-gradient(to top, var(--bg-color) 0%, transparent 100%)`。其底色使用的是 `body` 纸底色 `var(--bg-color)`（亮色下是暖纸白 `#F8F6F0`）。
  - 然而，详情面板 `.detail-panel` 已经使用 `.micro-mask-panel` 的半透明白底重构：`background: rgba(255, 255, 255, 0.8) !important`。
  - **冲突结果**：在折叠状态下，由于 `.fade-mask` 的底色是不透明的 `#F8F6F0`，而它叠在半透明的 `.detail-panel` 上，会导致遮罩下部呈现出一个**突兀、不透明的黄灰色“拼接缝脏色块”**，直接毁掉了卡片材质的通透感和毛玻璃悬浮效果。

### 4. 【中危】大字号下缩减 Padding 导致的文字贴边压迫感
- **现象描述**：
  - 重构代码将 `.detail-panel` 的 padding 压缩至 `1.8rem 1.2rem !important`，左右内边距仅为 `1.2rem` (19.2px)。
  - 当 `textScale = 1.5` 时，正文字号膨胀至 `1.65rem` (26.4px)。
  - **冲突结果**：文字的实际大小 (26.4px) 已经大于其与边框的物理距离 (19.2px)。按照版式布局原则，这会导致文字非常局促地“贴”在黛墨色边框上，造成视觉重心失衡和强烈的压迫窒息感。

### 5. 【高危】小字号下暗色模式对比度无障碍失效
- **现象描述**：
  - 当 `textScale = 0.8` 时，POI 描述文字 `.poi-card-info p` 字号缩小至 `0.72rem` (约 `11.5px`，低于中文阅读临界点)。
  - **无障碍冲突**：在暗色模式下，该小字号文字颜色为 `--text-secondary`（`#86868B`），而面板背景为 `rgba(28, 28, 30, 0.85)`。通过对比度计算得出，两者之间的**相对对比度仅为 3.71:1**，远远低于 WCAG 2.0 AA 规范规定的普通文本对比度最低标准（4.5:1）。结合 11.5px 的超小字号，这会导致这部分历史描述在暗色模式小字号下**几乎无法认读**。

### 6. 【中危】小字号留白增加与极简黛墨边框“隐形融化”
- **现象描述**：
  - 缩小时，文字体积缩小，而卡片的 padding（如 `.section-card` 垂直 `2rem`）仍保持原样，导致卡片内部留白大幅度增加，页面呈现较多空洞。
  - 为了极简风，重构将 `.micro-mask-panel` 的边框和阴影调得极其克制：`border: 1px solid rgba(29, 37, 32, 0.05) !important`。
  - **冲突结果**：在亮色模式下，米黄纸底色 `#F8F6F0` 和面板白色底 `rgba(255, 255, 255, 0.8)` 非常接近。这个只有 5% 不透明度的超细黛墨边框，在光线稍微强烈或屏幕稍弱的设备上，会**彻底隐形**。用户将完全看不到卡片的边界轮廓，导致卡片与背景“糊成一片”，极简极雅风退化为“惨白且结构混乱”。

## 🛠️ 深度修复与防御性重构建议 (Mitigations)

为了彻底解决极端字体缩放下的视觉崩塌问题，建议进行如下防守性修改：

1. **统一文字缩放层级（标题联动缩放）**
   - 不应对 `h3`, `h2` 使用固定字号。卡片内标题和正文应该**全部基于 `--text-scale` 变量按比例缩放**，或者对标题也追加 `calc`。
   - 例如：
     ```css
     .poi-card-info h3 {
       font-size: calc(1.3rem * var(--text-scale));
     }
     .section-card h2 {
       font-size: calc(2rem * var(--text-scale));
     }
     ```

2. **采用 `em` 或百分比联动容器高度与 padding**
   - 不要将折叠高度硬编码为 `200px`。应当改为使用基于当前字号的 `em` 或 `rem` 单位，例如 `max-height: 12.5rem`（在 1.5 倍下自动伸展为原来的 1.5 倍高），防止溢出和文字被横向物理裁切。
   - 同理，`.detail-panel` 的 padding 也应使用 `rem` 或随缩放自动适配。或者，设置 padding 时配合 `textScale` 进行 calc。

3. **动态修正首字下沉样式**
   - 首字下沉 `::first-letter` 的字号也应使用 `calc(3.5rem * var(--text-scale))` 进行缩放，并保证 float 后的 `line-height` 和 margins 可以根据 `textScale` 动态自适应。

4. **使用背景色混合修剪渐隐遮罩 `.fade-mask`**
   - `.fade-mask` 的渐变不应使用 `var(--bg-color)`，而是使用 `var(--glass-bg)`（即 `.micro-mask-panel` 同色底的透明渐变），或者将其设为：
     `linear-gradient(to top, rgba(255, 255, 255, 0.8) 0%, rgba(255, 255, 255, 0) 100%)`。
   - 暗色下联动：
     `linear-gradient(to top, rgba(28, 28, 30, 0.85) 0%, rgba(28, 28, 30, 0) 100%)`。
   - 这样才能完美消除拼接缝。

5. **增强极简边框与暗色文字对比度**
   - 提升小字号在暗色模式下的文字对比度。建议在 `--text-secondary` 降低字号时自动切换回稍亮的颜色，或者对小字号下的 opacity 做无障碍增强。
   - 将 `.micro-mask-panel` 的细边框在亮色下的不透明度从 5% 提高到至少 10% (`rgba(29, 37, 32, 0.1)`)，以保证在小字体大留白下边界轮廓依旧清晰可读。
