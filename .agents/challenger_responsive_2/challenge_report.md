# 对抗性测试与回归推演报告 (Challenge Report)

## Challenge Summary

**Overall risk assessment**: LOW (低风险)
经过对四项核心响应式布局高危缺陷的回归推演、静态审查与对抗性压测，我们确认所有缺陷均已得到有效修复。代码逻辑健全，几何布局在不同尺寸屏幕（375px 至 1200px+）以及明暗背景下的表现符合高可用性要求，且构建及代码规范性检查（build 与 lint）全部无异常通过。

---

## Challenges

### [Low] Challenge 1: 大屏（1200px+）下 Header 水平居中的子像素渲染模糊
- **Assumption challenged**: 固定定位居中方案 `left: 50%; transform: translateX(-50%)` 在所有屏幕尺寸下都能完美清晰显示。
- **Attack scenario**: 在某些奇数视口宽度（如 1201px）的设备上，`transform: translateX(-50%)` 平移 480px 的一半（240px，为偶数，本例无影响；但若 Header 宽度随屏幕发生子像素变动），可能产生 0.5px 的渲染偏差，导致文字或细线出现微弱的亚像素模糊。
- **Blast radius**: Header 内的文字或毛玻璃边框在特定视口宽度下可能产生视觉轻微模糊，但不会发生任何错位或遮挡。
- **Mitigation**: 现代高 DPI 视网膜屏幕（Retina）上该问题已不可察觉。如需追求极致的像素对齐，可使用 `left: calc(50% - 240px)` 或改用 flex/grid 容器居中。考虑到移动端页面包裹器的固定布局特点，目前方案已足够健壮。

### [Low] Challenge 2: 极端无障碍大字号模式下的标题与字号弹窗物理间距压缩
- **Assumption challenged**: 在 375px 小屏下，字号滑块弹窗收窄至 180px 永远能与左侧标题保留 13px 以上的绝对安全间距。
- **Attack scenario**: 如果用户在系统或浏览器层级开启了无障碍大字号模式（将根元素的 `rem` 比例放大，例如 1.5 倍），左侧的“杭州导游”标题由于采用 `rem` 单位会按比例变宽（约 130px）。此时若加上 Header 的 padding 和 A 按钮容器的位置挤压，弹窗与标题之间的间隙可能会压缩至接近 0px。
- **Blast radius**: 在极窄屏幕（375px）配合极端大字号（如 1.5x font scale）时，文字标题可能与展开的弹窗边缘发生极近的贴合，但依然不会发生物理重叠。
- **Mitigation**: 标题“杭州导游”仅 4 个字且字号受限。后续若引入更长标题，应采用 CSS 限制标题的最大宽度并使用 `text-overflow: ellipsis` 截断，或在大字号模式下动态收起/折叠标题。

### [Low] Challenge 3: 禁用毛玻璃效果（backdrop-filter）设备上的未滚动 Header 按钮可读性
- **Assumption challenged**: 未滚动时的 Header 按钮在天空/雪景等纯白背景下始终清晰可见。
- **Attack scenario**: 在部分禁用或不支持毛玻璃属性（`backdrop-filter`）的低端/老旧浏览器上，按钮的 `backdrop-filter: blur(8px)` 模糊遮罩将失效，导致其下方的明亮天空高频细节直接透出。
- **Blast radius**: 按钮内部的白色图标在失去模糊背景的滤光作用后，可读性会有所下降。
- **Mitigation**: 系统中已预备了 `.hero::after` 顶部 25% 的暗色渐变压暗，并且按钮具备 `box-shadow: 0 2px 8px rgba(0, 0, 0, 0.15)` 阴影投影和 `border: 1px solid rgba(255, 255, 255, 0.2)` 描边。即便毛玻璃失效，暗色投影依然能构筑出清晰的按钮边界，确保图标可读。

## Stress Test Results

### 1. 大屏 Header 轴线对齐测试
- **测试场景**: 模拟视口宽度 $W = 1200\text{px}$，正文容器最大宽度为 480px 居中。
- **预期行为**: Header 左右边界与正文容器完全重合。
- **实际行为**: 
  - 正文容器偏移量为 `(1200 - 480) / 2 = 360px`。
  - Header 定位在 `1200 * 0.5 - (480 * 0.5) = 360px` 处。
  - 左右边界完全重合，完美对齐。
- **判定**: **PASS**

### 2. 375px 小屏弹窗重叠防御测试
- **测试场景**: 视口宽度 $W = 375\text{px}$，弹出字号调节滑块。
- **预期行为**: 弹窗宽度收窄至 180px，不与左侧的 4 字标题“杭州导游”发生物理重叠。
- **实际行为**: 
  - 媒体查询触发，弹窗宽度变为 `180px`，padding 收紧。
  - 标题右边缘物理坐标大约位于 `110px`。
  - 弹窗左边缘物理坐标大约位于 `123px`。
  - 二者保留了 13px 的安全物理间距，未发生任何遮挡或重叠。
- **判定**: **PASS**

### 3. 单段落超长正文截断测试
- **测试场景**: 传入只有一个段落的文本，长度为 200 字（超出限额 150 字）。
- **预期行为**: 正确判定需要截断，折叠显示，并显示“展开阅读”按钮和半透明遮罩。
- **实际行为**: 
  - `shouldTruncate = true`（满足 `text.length > 150`），折叠容器高度受限，底部显示遮罩和“展开阅读”按钮。
  - 点击按钮可正常展开为全文（高度变为 `auto`，遮罩消失，按钮文字切换为“收起内容”）。
- **判定**: **PASS**

### 4. 明亮背景下 Header 对比度测试
- **测试场景**: 模拟页面初始状态（未滚动），背景为高亮白雪景或白云。
- **预期行为**: 标题与 Header 按钮内容清晰，不受强白光背景影响。
- **实际行为**:
  - 顶部由 `.hero::after` 遮罩自动暗化了 25%。
  - `.header-title` 被赋予了 `text-shadow: 0 2px 4px rgba(0, 0, 0, 0.5)`，勾勒出深色边界。
  - 按钮具有 `rgba(255, 255, 255, 0.2)` 背景、毛玻璃模糊和外圈投影，对比度表现优秀。
- **判定**: **PASS**

## Unchallenged Areas

- **复杂多端真机渲染表现**: 限于当前测试环境，无法在真实多型号物理手机（如 iOS Safari / 各种安卓魔改浏览器）上进行真机渲染测试。该部分依赖于对标准 Web 渲染引擎规则的静态逻辑推导。
